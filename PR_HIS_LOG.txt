When inspecting allocation events, the frames stack is dominated by PyTorch/C++ internals. There was no way to quickly identify which line of user Python code triggered the allocation, especially when profiling code running on SSH-accessed remote machines.

Changes
SSH config auto-detection
get_ssh_config_path() — resolves ~/.ssh/config cross-platform (USERPROFILE/HOMEPATH on Windows, Path.home() on Linux/macOS)
parse_ssh_config() — parses all Host blocks into { alias: { "user": ..., "hostname": ... } }; keys normalized to lowercase; * wildcard skipped
New UI row (SSH bar)
SSH Host dropdown — populated from ~/.ssh/config host aliases; first real host pre-selected on startup
Frame user filter text box — auto-filled with the selected host's User value; editable manually; updates details panel live
Frame filtering
get_user_frames(frames, username) — returns only .py frames whose path contains /username/ (POSIX) or \username\ (Windows)
Details panel
The details panel now shows a User Code Frames section above the raw JSON, so the allocation call site in user code is immediately visible without scrolling through 40+ runtime frames:
